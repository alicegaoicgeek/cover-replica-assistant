# 封面复刻助手 · Cover Replica Assistant

基于参考图二创——拆解成熟封面的通用设计结构，迁移到你的标题和品牌上。

## 核心理念

> 不要让 AI 凭空想封面，而是让它先拆一张成熟参考图，再把结构迁移到你的标题、人物和账号风格上。

## 速查

```
用户输入(对标图+标题+比例) → 决策节点(信息完整性检查) → 参考图拆解(7维度)
→ 版权边界过滤 → 模板填充 → Agnes适配 → 输出提示词 → 生图 → 评分迭代
```

## 版权边界（铁律）

以下内容**绝不复制**，仅提取通用设计结构：
- ❌ 具体人物形象、签名、模特搭配
- ❌ Logo、品牌标识、平台水印
- ❌ 原文案、独特装饰元素
- ❌ 植物摆放、背景装饰、独特识别元素

**可复刻的通用结构**：构图比例、标题位置与层级、字体气质、色彩氛围、视觉重心、画面密度。

## 工作流

### Step 1: 信息收集

向用户一次性收集 3 样：
1. **对标封面图**（上传或 URL）
2. **新标题文案**（主标题 / 副标题 / 标签）
3. **画幅比例**（3:4 / 4:3 / 1:1 / 9:16 / 2.35:1）

🛑 **决策节点**：信息不完整时——
- 缺复刻方式 → 确认：轻度借鉴 / 深度复刻（更贴近参考图结构）
- 缺新标题 → 让用户提供替换标题
- 完整 → 进入 Step 2

### Step 2: 参考图 7 维度拆解

用 agent vision 读参考图，拆解：

| 维度 | 拆解内容 |
|------|---------|
| 画面比例 | 实际宽高比 |
| 标题位置与层级 | 主标题在哪、多大、什么字体气质 |
| 主体/素材位置 | 人物或主体占画面比例、位置 |
| 字体气质 | 粗体/细体、衬线/无衬线、颜色、字距 |
| 色彩氛围 | 主色、辅色、色调冷暖 |
| 视觉层级 | 第一眼看什么、第二眼看什么 |
| 画面密度 | 留白多还是信息密集 |

### Step 3: 版权边界过滤

从 Step 2 的拆解结果中，去除不可复刻的元素，保留通用设计结构。

### Step 4: 提示词生成

根据复刻深度选择模板（见 `references/prompt-templates.md`），填充：
- 构图说明（从 Step 2-3 提取）
- 标题文字（用户提供）
- 字体说明（字重、颜色、大小、位置）
- 主体说明（用户提供或从参考图提取通用特征）
- 氛围说明（色彩、光影、背景）

### Step 5: Agnes 适配

```bash
# 配置（首次）
agnes-img-cli --configure

# 生成封面
agnes-img-cli -s 1024x1536 -n 1 "提示词"
# 或从 stdin
echo "提示词" | agnes-img-cli -s 1024x1536
```

**常用尺寸映射**：
| 画幅 | Agnes size |
|------|-----------|
| 3:4 | 1024x1536 或 768x1024 |
| 1:1 | 1024x1024 |
| 9:16 | 768x1365 |
| 2.35:1 | 1410x600 |

### Step 6: 评分迭代

生成后按 6 维度评分（每项 0-10）：

| 维度 | 标准 | 权重 |
|------|------|------|
| 构图 | 强视觉重心，不对称但有平衡 | 25% |
| 字体 | 粗体大字 overlay，无 AI 塑料感 | 25% |
| 色彩 | 主色+辅色≤3，对比鲜明，无彩虹渐变 | 20% |
| 主体 | 中央发光体/芯片/几何结构，清晰可辨 | 15% |
| 留白与密度 | 信息密度高但不拥挤 | 10% |
| 点击感 | 在消息列表里能停下来 | 5% |

**迭代策略**：
- 总分 < 6.0 → 重写提示词，调整构图和字体描述
- 总分 6.0-7.5 → 微调色彩和主体描述
- 总分 7.5-8.5 → 微调氛围和细节
- 总分 ≥ 8.5 → 通过，交付

每轮 agent QA 读图打分，根据扣分项针对性修改提示词，重新生成，直到收敛。

## 反模式

- ❌ 直接复制参考图的文案和人物
- ❌ 跳过版权边界过滤
- ❌ 不拆解参考图就直接写提示词
- ❌ 一次生成不满意就放弃（必须迭代到 8.5+）
